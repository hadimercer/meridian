BPMN Notation Legend

Start Event
Initiates a process. Green fill.

End Event
Terminates a process. Red thick border.

Intermediate Event
Double-border circle. Catches a trigger mid-process (e.g., joins via link).

Task
An atomic unit of work performed by a lane actor. Rounded rectangle.

XOR Gateway
Exclusive decision — exactly one path taken. Marked with ×.

AND Gateway (Parallel)
All outgoing paths execute simultaneously. Marked with +.

Sequence Flow
Normal process flow within a lane or pool. Teal solid arrow.

Message Flow
Cross-lane communication (e.g., System triggered by Owner action). Orange dashed arrow.

Loop / Conditional Path
Feedback loop or "No" branch from a gateway. Amber dashed arrow.

Swimlane
Defines responsibility boundary. Each lane is a distinct actor or system component.

BPMN Compliance Note: These diagrams use simplified BPMN 2.0 notation appropriate for a portfolio BA artifact. Formal BPMN 2.0 would require additional notation elements for event sub-processes, boundary events, and formal message correlation. The core constructs — sequence flows, message flows, gateways, events, tasks, and swimlanes — are all correctly applied. The diagrams are designed for stakeholder communication and developer handoff, consistent with BABOK v3 requirements elicitation artifacts.
